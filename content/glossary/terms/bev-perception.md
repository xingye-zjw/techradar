# BEV感知

**BEV（Bird's-Eye View，鸟瞰图/俯视视角）感知** 是自动驾驶领域的主流场景表示范式：将环绕车身的多摄像头、激光雷达、毫米波雷达等异构传感器输入，统一转换到「自车上方俯视」的正交视角特征空间中，再基于 BEV 特征进行 3D 目标检测、车道线检测、可行驶区域分割、动态占用网络（Occupancy Network）等下游任务。

## 为什么需要 BEV？

传统「各传感器独立处理 → 后融合」方案存在诸多问题：

### 痛点 1：2D 检测框无法承载真实 3D 信息

摄像头做 2D 检测 → 用单目深度估计 + PnP 投影回 3D。误差链叠加：检测框 ±5 像素就可能导致 10 米距离处 ±1.5m 定位误差，无法用于规划控制。

### 痛点 2：跨传感器「谁信谁」难题

摄像头检测到前方 50m 有卡车，毫米波雷达测出来是 45m，激光雷达检测到 48m。后融合时做简单加权？但不同距离、光照、遮挡下，每个传感器可靠性完全不同，规则越写越复杂。

### 痛点 3：无法自然利用多视角冗余

前方目标横跨了前视主相机和前左/前右广角相机的重叠区域，2D 检测会出两个框，需要额外做 NMS + 跨视角关联，流程割裂。

### BEV 统一范式的解决思路

> 在同一个「俯视视角特征平面」中，每个 (x,y) 位置对应的物理世界坐标是唯一、确定的。
> 无论来自哪个相机/雷达/激光，特征都投影到这同一个平面上相加/拼接/注意力融合。

这样一来：

- 下游规划模块直接拿 BEV 网格坐标即可（世界坐标系 = 自车局部坐标系）
- 传感器融合变成了「同坐标系下的特征融合」，模型自己学习不同条件下各模态权重
- 横跨多视角的目标天然拥有完整、无重复的 BEV 表示

## 三类视角变换方法深度解析

### 方法 1：LSS（Lift-Splat-Shoot，2020）——隐式深度分布式投影

LSS 是 BEV 几何方法的奠基之作，核心三步：

```
Step 1: Lift（抬升）
对图像特征图上的每个像素 (u,v)：
  预测 D 个离散深度候选 bin 的概率分布 P(d_1...d_D)
  生成 D 个 3D 点云特征：(u,v,d_i) → 内参 K → 相机坐标系 3D 坐标
  每个点特征 = F_pixel(u,v) × P(d_i)

Step 2: Splat（投影）
把相机坐标系 3D 点 → 外参 T → 自车坐标系 → 量化到 BEV 网格坐标
同一 BEV 网格落入多个点特征 → 累加（Pooling）

Step 3: Shoot（卷积）
对 X×Y×C 的 BEV 特征图做 3×3 Conv 平滑掉量化伪影
```

LSS 优势：纯几何，训练易收敛；劣势：深度估计不准时误差直接投射，「深度错 1m → BEV 错 1m」。

### 方法 2：BEVFormer（2022）——可学习注意力查询

BEVFormer 完全抛弃了显式深度估计，用 Transformer 注意力「学」出对应关系：

1. 预定义一组可学习的 **BEV Query**（尺寸 100×100×256，相当于一张「空的 BEV 特征画布」）
2. 每个 BEV Query 的 3D 参考点坐标 $(x,y,z)$ 已知（BEV 网格中心 + 3 个采样高度）
3. 投影到每个相机画面得到 2D 参考点 $(u,v) = K \cdot T \cdot (x,y,z,1)$
4. 对参考点邻域取 4×4 = 16 个采样位置，做 **Deformable Cross-Attention**
5. 同时对历史时刻的 BEV 特征做 **Temporal Self-Attention**（对齐自车运动补偿后），引入时序信息

BEVFormer 在 nuScenes 上用纯视觉达到 56.9% NDS，比 LSS 方案高 10+ 个点。其核心洞见：**显式深度不准 → 干脆用注意力隐式对齐**。

```python
# 伪代码：BEVFormer 的一次 Deformable Cross-Attention
bev_queries = torch.randn(B, 100*100, 256)
ref_3d = get_reference_points(H=100, W=100, Z=3)          # (10000, 3, 3) 3个高度
ref_2d = project_cam2img(ref_3d, lidar2cam, cam_intrins)  # (10000, N_cam, 3, 2)

for lvl, feat_lvl in enumerate(multi_scale_feats):   # 对多尺度图像特征
    attn_feat = deform_attn(
        query=bev_queries,
        value=feat_lvl.flatten(0,1),    # 把6路相机特征按通道拼
        sampling_loc=get_offsets(ref_2d, level=lvl),
        attn_weights=get_weights()
    )
    bev_queries = bev_queries + attn_feat
```

### 方法 3：View Former / GKT——MLP 全局映射

「几何投影和注意力都太麻烦，直接学一个从图像特征全局到 BEV 特征的映射函数不行吗？」

GKT（Graph-based Knowledge Transfer）和 PolarFormer 证明：只要数据量足够（>100万帧），简单的 Global-AvgPool + 2 层 MLP 也能学到不错的全局→BEV 映射，再用图卷积在 BEV 网格间传播信息做细节补全。这类方法模型小、速度快，但对离群场景泛化性略差。

## 多模态 BEV 融合策略

纯视觉 BEV 受光照、雨天、远距离深度估计限制，工业落地一定是 **Camera + LiDAR + Radar 三模态融合**：

| 融合阶段                   | 方法                                                                                                                              | 优劣                                                      |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| **前融合**（数据级）       | 在点云阶段就把图像像素特征贴到点云上（PointPainting 类）                                                                          | 保留细粒度信息，但计算量大，跨模态标定误差直接影响        |
| **中融合**（特征级，主流） | Camera 分支出图像 BEV、LiDAR 分支出点云 BEV、Radar 分支出雷达 BEV → 三路拼接 + Cross-Modal Transformer / ConvLSTM 融合 → 统一 BEV | 兼顾效率和效果，工业界首选（Tesla FSD、小鹏 XNGP 都用此） |
| **后融合**（结果级）       | 各模态单独做 3D 检测 → 3D NMS + Dempster-Shafer 证据融合                                                                          | 实现简单、模态独立可控，但无法解决双重误检/漏检           |

中融合的关键：**特征对齐 + 动态加权**

- 图像 BEV 的 (x,y) 像素和 LiDAR BEV 必须物理位置精确对应（标定误差会导致特征错配）
- 用一个小型门控网络（Sigmoid 输出 0-1 权重）根据场景动态加权：夜间图像权重从 0.5 降到 0.1，雨雪天雷达权重从 0.2 升到 0.5。

## 下游任务头设计

### 1. 3D 目标检测（CenterPoint 头）

直接在 BEV 特征上做密集预测：

- 热图头：每个 BEV 网格预测该位置是否为目标中心（Heatmap，Focal Loss）
- 回归头：目标尺寸 (w,l,h)、朝向 yaw、高度 z、速度 (vx, vy)
- 后处理：NMS → 3D 框

### 2. 车道线检测

BEV 空间车道线天然无遮挡（俯视视角），用 3 阶贝塞尔曲线或 Polyline 回归：

- 每个车道输出 N 个控制点（N=11）+ 车道类型（实线/虚线/双黄等）

### 3. 占用网络（Occupancy Grid，Tesla Optimus 同款）

将 3D 空间体素化为 V×X×Y×Z 网格，每个体素预测「被占用概率 + 语义类别」。相比 3D 检测只输出目标框，占用网络能表示护栏、锥桶、散落物、施工围挡等非标准障碍物。

## 训练工程要点

### 1. 数据增强对 BEV 至关重要

BEV 模型非常容易过拟合特定道路布局，必须强增强：

```python
# 图像侧：
A.RandomBrightnessContrast(0.3, 0.3),
A.MotionBlur(blur_limit=7, p=0.2),   # 模拟车辆颠簸
A.CoarseDropout(max_holes=8)         # 模拟镜头脏污

# BEV 侧（极其关键！）：
bev_rot = random.uniform(-22.5°, 22.5°)   # BEV 特征随机旋转
bev_scale = random.uniform(0.9, 1.1)       # BEV 特征随机缩放
bev_flip = random.random() < 0.5           # 左右翻转（连同所有 3D 框标签）
```

> 业界共识：**BEV 任务中，合理的 BEV 空间增强 + 标定扰动可以让 NDS 涨 5 个点以上**。

### 2. 多任务联合训练

检测、车道线、占用三个头共用 BEV 编码器，用多任务 Loss 联合训练：

```
L_total = (
    1.0 * L_det_heatmap      # 检测热图 Focal
  + 0.5 * L_det_regression   # 检测框回归 L1 + GIoU
  + 0.3 * L_lane_bezier      # 车道控制点 L1
  + 0.4 * L_occupancy_ce     # 占用体素 CE
)
```

多任务训练会互相促进性能（检测任务学到的车辆边缘特征，对占用任务的车辆轮廓分割有迁移）。

### 3. 预训练策略

纯 BEV 数据成本极高（激光雷达标注 1 元/帧）。先用 **OpenLaneV2 / ONCE / NuPlan 等开源数据预训练 20 epoch → 自有数据微调 5 epoch → DPT 蒸馏对齐**。这套组合可以让 10 万帧自有数据达到 50 万帧从头训练的效果。

## 部署与端侧优化

BEV 模型计算量大（6 路 1280×720 输入 + BEVFormer 注意力），端侧部署需要做三级优化：

### Level 1：算子优化（TensorRT / CUTLASS）

- Deformable Attention 自定义 CUDA Kernel 替换原生 PyTorch 实现，延迟降 60%
- BEV Pooling 用 Atomic Add 写 CUDA Kernel，避免 scatter 循环

### Level 2：量化蒸馏

- 先做 FP16 部署（无损），再用 QAT（量化感知训练）做 INT8：
  - 图像骨干做 INT8（精度损失 < 1 NDS）
  - BEV Queries 和注意力保留 FP16（量化掉点大）
- 用大模型蒸馏小模型：BEVFormer-L（200FPS 卡）→ BEVFormer-S（200TOPS 车端 Orin 上 25 FPS + NDS 掉 2）

### Level 3：流水线并行

将 6 路相机预处理、骨干推理、视角变换、检测头解耦到 Orin 不同 GPU Stream + DLA 硬件上并行跑，端到端 20ms（50 FPS）。

## 未来方向

1. **端到端 BEV 到规划**：目前 BEV 感知输出 3D 框 → 预测模块预测轨迹 → 规划模块算路径。新一代架构（如 UniAD、AD-Kit）直接把 BEV 特征送进 Planner 端到端训练，用 imitation learning + RL 学出「感知即规划」。
2. **4D BEV（时空统一）**：把过去 N 秒的 BEV 堆叠成时空 4D 体，用 4D Convolution 直接学习动态目标运动轨迹，不需要单独的跟踪模块。
3. **跨城域泛化 + 地理先验**：给 BEV 模型注入 HD Map 先验（某位置肯定是车道、某位置是绿化带），与实时观测做贝叶斯融合，解决极端天气感知失效问题。
4. **从自动驾驶走向机器人**：人形机器人（Optimus）、工业机械臂的「世界模型」表示，本质也是 3D BEV 视角——BEV 技术正在从自动驾驶赛道走向整个具身 AI 领域。

BEV 感知的成功，本质是一次「坐标系思维」的胜利：通过选择一个对下游任务最友好的统一中间表示，把原本割裂的多传感器、多任务、多模态给「串」了起来。这套方法论未来在自动驾驶之外的具身智能、三维重建、AR 领域还会被反复验证。
