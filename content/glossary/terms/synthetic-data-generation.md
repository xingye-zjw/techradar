# 合成数据生成

**合成数据生成（Synthetic Data Generation, SDG）** 是通过算法、仿真器、生成式模型等人工手段创造训练数据的技术，用于弥补真实数据获取难、标注贵、分布不均衡、隐私受限等痛点。Gartner 预测到 2030 年，大模型训练所用数据中 **合成数据占比将超过真实数据**，成为 AI 模型迭代的主要燃料。

## 为什么合成数据越来越重要？

真实数据面临四重天花板：

| 天花板         | 说明                                                                                    |
| -------------- | --------------------------------------------------------------------------------------- |
| **成本天花板** | COCO 级目标检测数据集标注 1 张约 ¥5；医疗影像/自动驾驶 1 张 ¥50~200；100 万张即亿级投入 |
| **长尾天花板** | 真实世界是长尾的，「凌晨暴雨天逆光下骑自行车穿雨衣的外卖员」等极端场景几乎采不到        |
| **合规天花板** | GDPR / 个人信息保护法 / 医疗 HIPAA 使得人脸、病历、金融数据不敢随便用                   |
| **迭代天花板** | 模型一周迭代一个版本，但真实数据采集 + 标注周期动辄 3 个月，永远跟不上                  |

合成数据完美绕开这些：

- 想要多少生成多少，边际成本 ~0（GPU 电费）
- 任意长尾场景可控生成，极端光照、罕见事故、稀有病症
- 0 隐私风险（所有样本都是假的，不对应真实个人）
- 模型迭代到 v3，立即造对应的数据 v3 反哺训练，形成闭环

## 四大领域合成数据技术栈

### 领域 1：计算机视觉（CV）——3D 仿真 + 程序化生成

CV 合成数据是最成熟的赛道，已经在工业落地 5+ 年：

**工具链**：

| 层级         | 工具                                                           | 作用                                 |
| ------------ | -------------------------------------------------------------- | ------------------------------------ |
| 高端渲染     | Unreal Engine 5 + MetaHuman、Unity HDRP                        | 影视级真实感，用于目标检测/分割      |
| 轻量程序化   | Blender Python API、Trimesh、Pyrender                          | 批量化生成简单几何场景，速度快       |
| 专用仿真     | Carla（自动驾驶）、Isaac Sim（机器人）、Vega（医疗）、SimReady | 内置物理引擎 + 传感器模型 + 标注 API |
| 域迁移风格化 | Stable Diffusion + ControlNet + IP-Adapter、GAIA、TIPO         | 把「仿真风」转成「真实风」           |

**自动驾驶合成数据流水线示例**：

```
┌─────────────────────────────────────────────────────────────┐
│  1. 场景程序化生成（Blender / Carla Python API）               │
│     - 随机天气：晴朗 / 阴天 / 暴雨 / 小雪                       │
│     - 随机时段：日出 / 正午 / 黄昏 / 深夜                       │
│     - 道路几何：直道、S 弯、坡度、施工车道变窄                   │
│     - 演员放置：N 辆车（车型×100）、行人×50、骑行者×10          │
│     - 随机插入「长尾场景」：前车掉落货物、路面积水、流浪狗横穿    │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  2. 多传感器渲染 + 自动标注                                    │
│     - 6 路环绕相机（2048×1556） + 去畸变 + 镜头光晕 + 噪点     │
│     - 64 线激光雷达（模拟距离噪点、雨滴回波假点）               │
│     - 标注直接从引擎拿：2D/3D 检测框、实例分割掩码、深度、法线   │
│     → 一晚上渲染 50 万帧，标注成本 ¥0                          │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  3. 域随机化（Domain Randomization，核心！）                    │
│     「让合成数据千差万别，让模型被迫学真正的物理规律而非记住画风」 │
│     - 材质：同一款车，车漆反射率 × 50 种随机                    │
│     - 光照：太阳高度角 + 方位角 + 色温全随机                     │
│     - 相机：曝光 ±2EV、ISP 随机锐化/模糊、镜头呼吸效应           │
│     - 位置：演员位置 ±0.5m 抖动、障碍物尺寸缩放 0.7~1.3          │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  4. 风格迁移（Sim2Real Gap 最后一跳）                           │
│     ControlNet + 真实数据参考图：把合成道路「沥青颜色」         │
│     「植被分布」「天空色调」对齐真实采集道路风格，              │
│     保留几何结构和标注不变                                      │
└─────────────────────────────────────────────────────────────┘
```

仅用这种「100% 合成数据」训练 YOLOv8，在真实 nuScenes 验证集上车辆检测能达到真实数据训练模型 **92%** 的精度。再用 10% 真实数据微调 → **99% 精度**，标注成本省 90%。

### 领域 2：自然语言处理（NLP）——强老师模型 + 进化指令

大模型时代，合成数据已经在重塑 LLM 训练范式：

**Self-Instruct（斯坦福 2023）**：

> 用强模型（GPT-4）根据少量人工种子指令，自我扩展生成海量指令数据，用来训练弱模型（Llama、Mistral）。

Self-Instruct 流水线：

```python
seeds = load_json("人工写的 175 条高质量种子指令.json")

for step in range(100):
    # 每次从种子中挑 8 条放入 prompt 作为 in-context 示例
    prompt = compose_prompt(sample(seeds, k=8), new_task_prompt)
    response = gpt4.generate(prompt, temperature=0.8)  # 高温度=多样性

    # 解析出模型「自编」的新指令
    new_instructions = parse_instructions(response)

    # 过滤低质量 + 去重（和已有数据 ROUGE-L 相似度 < 0.7 才保留）
    new_instructions = filter_and_dedup(new_instructions, seeds)

    seeds.extend(new_instructions)
    # 循环 → 175 条 → 1 周后变成 8 万条
```

**Evol-Instruct（WizardLM 2023）**：

> 对简单指令做「深度进化」，从浅到深生成难度梯度数据：
>
> - 加约束：「请用代码写出」→「请用 PyTorch 分布式 API + 异步队列写出，且必须加类型标注」
> - 加深推理：「求 1+2=?」→「小明有 1 个苹果，妈妈又给了他 2 倍于他原本有的……请用思维链分步回答」
> - 复杂化输入：「写个快速排序」→「给定一个亿级行 CSV 文件，每行一个整数，写一个外存不超过 2GB 的快速排序」

用 Evol-Instruct 合成的 7 万条数据微调的 Llama-2 → WizardLM，在 MT-Bench 上超过原 Llama-2 **200+ 分**，一度追平 GPT-3.5。

### 领域 3：结构化数据（表格/时间序列）

金融风控、推荐系统、用户增长场景中，用户行为表通常有严格的隐私合规：

**方法**：

- **CTGAN / TVAE**（SDV 开源库）：对表格列的联合分布建模，生成和真实表统计分布一致、但每行都是虚拟用户的假数据
- **DoppelGANger**：对时间序列（用户 APP 行为流、股票 K 线）做条件 GAN，保留周内/日内周期性和趋势
- **Diffpriv 差分隐私**：在真实数据统计量上注入可控噪声后再生成，从数学上保证无法反推任何个体

### 领域 4：强化学习（RL）——仿真器作为合成数据源

机器人、游戏、推荐系统的 RL 训练不可能让智能体在线探索（会炸）：

- 机器人：MuJoCo、Isaac Gym 用 GPU 并行跑 4096 个机械臂在仿真里学抓取 → 1 天跑完真实世界 1000 年经验
- 自动驾驶：CARLA 在仿真里制造 1 亿种「鬼探头」corner case 让 Planner 学避障
- 推荐系统：RecSim / RecBole-Gym 用环境模型模拟用户点击反馈，离线训练 RL 排序策略，再 A/B 上线

## 核心难题：Sim-to-Real Gap（仿真到真实的鸿沟）

「仿真训练模型一到真实场景就傻了」是合成数据最痛的问题。根本原因：**合成数据分布 P_sim(x,y) ≠ 真实分布 P_real(x,y)**。

解决方向三大流派：

### 流派 A：域随机化（Domain Randomization）

> 「不要把仿真做得像真实，而是把仿真做得足够多样，让真实变成仿真的一个子集」

极端案例：OpenAI 解魔方的 Shadow Hand 机器人——完全没给任何真实手的数据，只在仿真里把手的摩擦力、手指尺寸、线缆张力、摄像头角度随机化到离谱的范围，模型学到了「不管什么手都能解魔方」的通用策略，迁移真实机手一次成功。

### 流派 B：域适应（Domain Adaptation / Domain Generalization）

> 「学习一个特征空间 F，在这个空间中 P_sim(F(x)) 和 P_real(F(x)) 距离最小」

- 对抗训练（DANN）：加一个域判别器，特征提取器要骗过判别器（让它无法判断特征来自仿真还是真实）
- 风格迁移（CycleGAN / AdaIN / ControlNet）：把图像的「画风」从仿真转到真实，保持语义和标注不动

### 流派 C：数据混合混合（Hybrid Training）

> 「没必要 100% 合成，90% 合成 + 10% 真实 = 100% 真实的效果」

工业界黄金配比：合成数据（大而全，覆盖长尾）占 80~~90%，真实数据（精而准，保证核心场景）占 10~~20%。这在检测、分割、OCR 场景都已经被反复验证。

## 最新进展：用大模型 + 生成模型做合成闭环

2024 年起合成数据进入 **Data-Centric AI 2.0 时代**——模型自己发现弱点 → 自动合成针对性数据 → 训练模型新版本 → 再找新弱点，形成无人值守的数据飞轮：

```
┌──────────────┐  ① 模型弱在哪儿？     ┌────────────────────┐
│  VLM v1 模型  │ ──────────────────→  │  Agent 找失败案例    │
└──────┬───────┘   MMBench 跑评测      └──────────┬─────────┘
       │  ④ 训练新版本                           │ ② 定位问题类型
       ▼                                         ▼
┌──────────────┐   ③ 针对性合成          ┌────────────────────┐
│  VLM v2 模型  │  ←──────────────────  │ 数据合成 Agent       │
└──────────────┘   GPT-4V + SD3 生成    │ 「VQA 里蓝色小物体」 │
                                        │  识别率低，造 5000 张│
                                        │  蓝色小物体 VQA 对子 │
                                        └────────────────────┘
```

Google DeepMind 的 _Synthetic Data Helix_ 工作证明：在数学推理任务上用这种合成飞轮迭代 5 轮，模型 GSM8K 分数从 50% → 80%，全程没用额外真实标注。

## 未来展望与伦理

1. **合成数据质量自动评测**：现在合成数据质量靠「训一把看看精度」。未来会有像 perplexity 一样的自动化指标（如和真实数据的 FID、Precision/Recall 分布距离、训练动态特征一致性）直接预测「这组合成数据能不能涨点」。
2. **数据知识产权**：用米老鼠 3D 模型合成的数据训练模型算不算侵权？用受版权保护小说喂 Self-Instruct 生成的指令数据是否合规？合成数据的版权边界是未来 3 年法律热点。
3. **Deepfake 双刃剑**：合成数据生成技术和 Deepfake 技术是共通的，人脸/语音合成数据滥用（诈骗、抹黑）的防护技术（水印、溯源）必须同步发展。
4. **AGI 的唯一数据来源**：当 LLM 已经吃遍全网真实数据后，**下一轮能力跃迁只能靠合成数据**——智能体在仿真世界里做实验、学编程、搞科研，把自己的「经验」当作合成数据自举提升。

合成数据是 AI 从「数据采集者」进化为「数据创造者」的关键转折点。未来最强的模型，不是拥有最多真实数据的那个，而是拥有**最聪明的合成数据引擎**的那个。
