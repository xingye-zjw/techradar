# 语音合成-TTS

**TTS（Text-to-Speech，文语转换 / 语音合成）** 是将文字文本自动转换为自然、流畅、可理解的人类语音音频的技术。ASR（语音识别）是让机器「听懂人话」，而 TTS 则是让机器「学会说人话」——两者构成了人机语音交互的完整闭环。

TTS 的应用场景在大模型时代爆发式增长：

- **AI 助手**：ChatGPT 语音对话、豆包、文心一言语音模式
- **有声书 / 播客**：长文本一键转主播级语音（听小说、听新闻）
- **AI 配音**：短视频口播、广告配音、影视译制片配音
- **客服 / 外呼**：电商 AI 外呼、运营商/银行语音通知
- **个性化语音**：明星/家人声音克隆，「孩子睡前听爸爸妈妈讲故事」
- **具身机器人 / 智能眼镜**：真人一样的语音输出，是智能体「拟人感」的关键

## TTS 技术演进：从拼接合成到端到端大模型

### 第一代（~2015）：拼接合成 + 参数合成

| 类型                                            | 原理                                                                                                             | 优点                                           | 致命缺点                                                                          |
| ----------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- | ---------------------------------------------- | --------------------------------------------------------------------------------- |
| **拼接合成（Unit Selection）**                  | 专业播音员录 10~50 小时语音 → 切成音素/半音节/音节小片段 → 合成时动态选最合适的片段拼接起来，加上 WSOLA 时间调整 | 音质高，和真人几乎无差（因为片段都是真人录的） | 拼接处有断裂感；语气单一；语料库巨大；无法做跨语言/情感；换个说话人重新录几十小时 |
| **参数合成（Statistical Parametric，HMM/DNN）** | 用 HMM 或 DNN 建模文本特征→声学特征（频谱、基频 F0、时长）→ 用声码器（STRAIGHT / WORLD）重建波形                 | 平滑，不会断；语料需求小                       | 音质发闷、「机器味」浓、听起来像机器人说话，不自然                                |

用户体感：**10 年前的车载导航「前方左转」那种机器人声音，就是这一代。**

### 第二代（2017~2023）：两阶段深度学习 TTS（声学模型 + 声码器分离）

**核心思想：「把难的问题拆成两个分别解决」**

```
文本 Text (Phoneme / Character)
    │
    ▼
【阶段一：声学模型 Acoustic Model】（主流：Tacotron 2 / FastSpeech 2 / VITS）
    输入文字 → 输出「梅尔频谱 Mel-Spectrogram」（80 维 × T 帧的图像）
    决定：读什么字、读多快、音多高、哪里停顿、语气感情
    │
    ▼
【阶段二：声码器 Vocoder】（主流：WaveNet / WaveGlow / HiFi-GAN / WaveRNN）
    输入「梅尔频谱」→ 输出「采样率 22050/24000Hz 的原始波形 wav」
    决定：音质清不清、有没有杂音、像不像真人嗓子
```

这一代是工业落地的绝对主力，贡献了目前 95% 以上商用 TTS 系统。下面详解关键架构：

#### 声学模型里程碑

##### Tacotron 2（Google 2017，品质里程碑）

```
文本字符嵌入
    ↓
Encoder (3 层 BiLSTM) → 文本隐表示
    ↓
Attention（Location-Sensitive）+ Decoder (2 层 LSTM)
    ↓
逐帧自回归输出 80 维梅尔频谱（Post-Net 5 层 CNN 残差精修）
```

- 自回归 + 注意力 + Post-Net，第一次让 TTS 合成声音 MOS（主观平均意见分，5 分满分）突破 **4.5 分**（真人录音 MOS≈4.7）
- 缺点：自回归慢（一秒语音对应几百帧，一帧一帧吐）→ 1 秒语音 0.3 秒生成（RTF 0.3），实时并发有限

##### FastSpeech 2（微软 2020，速度里程碑——非自回归！）

> 既然自回归太慢，「能不能一次并行输出所有帧？」

创新点：

1. 去掉了自回归循环，用 Length Regulator（长度调节器）直接预测每个音素持续多少帧
2. 显式建模 F0（基频）、能量、时长三个韵律特征，不再靠注意力隐式学
3. 所有帧并行生成 → **Transformer Encoder 一把梭**

效果：

- 音质和 Tacotron 2 差不多（MOS 4.4 vs 4.5，几乎听不出差）
- **速度提升 270 倍！** 10 秒文字 → 10ms 生成完（RTF 0.001），1 张 T4 卡并发 500 路没问题
- 是工业在线合成的首选架构，**目前百度/阿里/讯飞线上主引擎基本都是 FastSpeech 2 及其变种**

##### VITS（2021，开山之作——变分自编码器 + GAN + 归一化流，声码器合一）

VITS 首次把「声学模型 + 声码器」两个阶段合一训练：

- 用 VAE 先验 + Normalizing Flow 建模文本 → 隐变量 → 波形
- 中间又加了 HiFi-GAN 判别器对抗训练，音质更逼真
- 端到端一体化，不用单独训声码器了，**第一次端到端 E2E 达到了两阶段架构的音质**
- 也是目前开源社区最火的架构，**大多数开源 TTS（Coqui TTS、GPT-SoVITS、ChatTTS）都基于 VITS 改进**

#### 声码器里程碑（从慢到快，从差到像）

| 声码器       | 年份 | 架构                               | 音质                              | 速度（单音频秒实时）                                    | 工业部署现状                                                  |
| ------------ | ---- | ---------------------------------- | --------------------------------- | ------------------------------------------------------- | ------------------------------------------------------------- |
| WaveNet      | 2016 | 自回归膨胀 CNN                     | ⭐⭐⭐⭐⭐                        | ❌ 慢死（几秒钟生成 1 秒）                              | 研究用，不部署                                                |
| WaveGlow     | 2018 | Flow-based 流模型                  | ⭐⭐⭐⭐⭐                        | ⭐⭐                                                    | 旧系统用                                                      |
| **HiFi-GAN** | 2020 | 多周期多尺度判别器 GAN             | ⭐⭐⭐⭐⭐（几乎和 WaveNet 没差） | ⭐⭐⭐⭐⭐ **快到飞起**（RTF 0.001 级，实时并发上千路） | **工业部署绝对王者，90% 线上 TTS 现在用 HiFi-GAN 或它的变种** |
| WaveRNN      | 2018 | 自回归 RNN + 稀疏化                | ⭐⭐⭐⭐                          | ⭐⭐⭐                                                  | 端侧小模型用                                                  |
| BigVGAN      | 2022 | HiFi-GAN + big filter + snake 激活 | ⭐⭐⭐⭐⭐（+0.05 MOS）           | ⭐⭐⭐⭐                                                | 新一代旗舰声码器                                              |

### 第三代（2023 以后）：零样本语音克隆 + 大模型 TTS 爆发

大模型 + VITS 带来了两个质变：**少样本/零样本声音克隆** 和 **自然度超越真人（在部分测试中）**

#### 里程碑 1：VALL-E（微软 2023 年 1 月——3 秒克隆！）

> 「给我 3 秒目标说话人的参考音频，我能模仿他/她的声音、语气、情感读任何文字。」

架构：基于 VQ-GAN + 自回归语言模型（类似 GPT），先把音频离散化为 1024 个声学 code → 用 6 亿参数 Decoder-only Transformer 学习「文字 + 声学 Prompt → 下一个声学 code」的条件语言模型。

VALL-E 之后声音克隆门槛直接降到 0。

#### 里程碑 2：ChatTTS（开源 2024——中文闲聊级自然度）

国内开源团队出品，专门针对**口语化聊天场景**优化：

- 自然度 MOS 4.6+，加入笑声、喘气、停顿、语气词（「嗯…那个…对吧？」这种人类日常说话的自然细节）——不像传统 TTS 朗读腔那么重
- 支持情感控制（开心/悲伤/愤怒）、语速控制、停顿控制
- 社区生态爆炸，出现 ChatTTS-Enhanced、CosyVoice（阿里）、FireRedTTS 等衍生

#### 里程碑 3：Fish Speech / Sparkles TTS（2024 下半年——开源顶级质量）

社区出现了第一批发声质量追平商业 API（ElevenLabs）的开源项目，支持：

- 多语言混合（同一句中英混读不换声）
- 高保真歌唱 + 语音混合
- 情感强度、语速、年龄、性别连续插值控制
- RTX 3090 实时 RTF < 0.1

## TTS 关键指标 & 主观评测

### 客观指标（只能做参考，主要还是靠人耳听）

| 指标                               | 含义                             | 说明                             |
| ---------------------------------- | -------------------------------- | -------------------------------- |
| **MCD（Mel Cepstral Distortion）** | 合成语音和真人录音梅尔倒谱的距离 | 越低越好，< 6dB 算优秀           |
| **MCD-T**                          | 时长归一化后的 MCD               | 更准确衡量音质                   |
| **F0 RMSE**                        | 基频 F0（音高）均方根误差        | 衡量调子准不准                   |
| **RTF（Real-Time Factor）**        | 合成耗时 / 音频时长              | 越短越好，在线场景 < 0.1         |
| **MOS 预测模型（UTMOS / NISQA）**  | 用神经网络自动预测人 MOS 分      | 比纯客观接近人类主观，但仍有偏差 |

### 主观指标（金标准，掏钱购买的依据）

**MOS（Mean Opinion Score，平均主观意见分）**：

> 找 20~~50 名被测试者，随机听若干合成语音样本（混有真人录音），用 1~~5 分打分：
> 5 = 完全听不出是合成的；4 = 略有机器感但完全不影响理解；3 = 有明显机器感；2 = 很难听懂；1= 完全听不懂
> 最后去掉最高分最低分，求平均分。

现在顶级 TTS 的 MOS 分数：

- 真人专业播音（Ground Truth）：4.6 ~ 4.8
- ElevenLabs / Fish Speech v1.5：**4.5 ~ 4.65**（已经非常接近真人）
- 开源 ChatTTS / CosyVoice：**4.2 ~ 4.4**
- 旧 FastSpeech 2：3.9 ~ 4.1
- 十年前 HMM 参数合成：2.5 ~ 3.0

**SMOS（相似性 MOS）**：克隆声音专用，0~5 分衡量「合成声音和目标人声音有多像」。

- 顶级方案（3 秒参考音频 + 大模型）：**4.2+**（普通用户基本听不出差异）

## TTS 快速上手：ChatTTS（开源中文首选）

```bash
# 安装 ChatTTS
pip install chattts modelscope nltk

# 下载预训练权重（~1.2GB，第一次自动下）
```

```python
import ChatTTS
import torch
import torchaudio

chat = ChatTTS.Chat()
chat.load(compile=False)  # compile=True 会用 torch.compile 加速（首次慢，后续快）

# ============ 基础：文字转语音 ============
texts = [
    "大家好呀，今天给大家介绍一下 TTS 语音合成技术在 AI 时代的最新进展～",
    "首先呢，我们先回顾一下 TTS 从拼接合成到端到端大模型的三代演进历程，然后再聊聊声音克隆的未来方向。",
]

# 控制参数（关键！！）
params_infer_code = {
    'spk_emb': torch.load("custom_speaker_emb.pt"),  # 【可选】加载特定说话人 embedding（音色）
    'temperature': 0.3,        # 温度越低越稳定，越高变音越多（0.3 合成稳定，0.8 即兴感强）
    'top_P': 0.7,              # 核采样，限制候选词累计概率
    'top_K': 20,               # 候选限制
    'prompt': "[speed_5][oral_2][laugh_0][break_6]",  # ChatTTS 特色控制 token：
    # speed 速度 0~9；oral 口语化程度；laugh 笑声频率；break 停顿程度
}

# 控制文本前后插入停顿/情绪
params_refine_text = {
    'prompt': '[oral_2][laugh_0][break_6]'
}

wavs = chat.infer(
    texts,
    use_decoder=True,
    params_infer_code=params_infer_code,
    params_refine_text=params_refine_text,
)

# 保存音频（torchaudio 保存为 wav，24kHz, 单声道, float32）
for i, wav in enumerate(wavs):
    torchaudio.save(f"output_{i}.wav", torch.from_numpy(wav).unsqueeze(0), 24000)

# ============ 进阶：3 秒少样本声音克隆 ============
# 步骤：
# 1. 用 3~10 秒目标说话人清晰音频（无噪音、纯人声）
reference_wav = "speaker_A_reference_5s.wav"
# 2. 用 ChatTTS 内部 SpeakerEncoder 提 embedding
spk_emb = chat.sample_random_speaker()  # 或者从参考音频算
# 3. 把 spk_emb 塞到 params_infer_code['spk_emb'] 再 infer

# 或者用社区 GPT-SoVITS 做高质量克隆（专业推荐，10 秒数据 + 微调 10 分钟，相似度更高）
```

## 工业级 TTS 系统工程优化清单

### 1. 文本前端（Text Frontend）——「先把文字写对，不然合成再牛也白搭」

原始文字到 TTS 模型输入的中间处理，这是**最容易忽略但影响 MOS 最大的模块**：

```
用户输入原始文本：
"我们公司Q3营收同比+12.3%，达到¥1,234万（约合$180万），SKU#7823将于10/1发布，联系400-888-9999转8023"
    ↓
【文本归一化 Text Normalization（TN）】→ 把符号展开成口语文字：
"我们公司第三季度营收同比增长百分之十二点三，达到人民币一千二百三十四万元（约合一百八十万美元），商品编号七八二三将于十月一日发布，联系四零零杠八八八杠九九九九转八零二三"
    ↓
【G2P 字音转换（Grapheme-to-Phoneme）】→ 标发音 + 多音字处理：
"我(wǒ3)们(men5)公(gōng1)司(sī1)第(dì4)三(sān1)季(jì4)度(du4)...（银行háng2，不行xíng2，长cháng2短/长zhǎng3大...）"
    ↓
【韵律预测 Prosody】→ 在哪里断句、哪里重读、哪里停顿
→ 加标点韵律标记 「#1 短停顿/#2 中停顿/#3 长停顿/#4 句末」
    ↓
送进声学模型
```

文本归一化 + G2P 做不好，TTS 模型再牛都没用——「¥1,234万」合成出来读成「人民币逗号一千二百三十四万」就凉了。商业 TTS 公司 70% 人力其实投在文本前端。

**工业推荐**：用 WeTextProcessing（NVIDIA NeMo 出品）+ THCHS-30 字典 + 定制领域正则，中文 TN 覆盖 99% 场景。

### 2. 情感 / 说话人控制

现代 TTS 不是只有一种声音，必须可控：

- **Speaker Embedding**：一个 d=256 向量 = 一种音色。想有 100 种音色 = 存 100 个 embedding（只占 100KB，比存 100 个单独模型省死了）
- **情感控制**：在 embedding 上连续插值，怒 ←→ 喜 之间任何中间情绪都能合成
- **语速、音量、语调**：F0 / energy scaling 直接调，无需重训

### 3. 流式合成 + 低延迟（对话必备）

AI 语音助手对话场景，用户一句话说完 → AI 要秒开口，不能等一整句 5 秒文字合成完再播放：

```
文字流式分段："我帮你 / 查一下 / 你这笔 / 订单 / 的 / 物流状态 / 是"
                ↓  每段 2~3 个字触发一次合成
流式声学模型（FastSpeech 2 改流式版本）
                ↓  每 200ms 输出一段梅尔频谱
流式 HiFi-GAN 声码器
                ↓  每 50ms 一帧音频块
WebSocket 推播放器 → 用户听起来「AI 在边想边说」，体感延迟 < 300ms
```

这套流水线，RTX 4090 单卡可稳定服务 80~100 路并发对话。

### 4. GPU 集群部署 + 成本优化

```
NLP / LLM Gateway → 文字结果出来
      │
      ▼
【TTS API Gateway K8s Service（10 台 16C）】
      负载均衡 + LRU 缓存（相同文字 + 说话人 → 命中缓存直接返回音频，省 99% 计算）
      │
      ▼
【TTS GPU Pool（30×A10 24G）】
      每台：文本前端（CPU）+ FastSpeech 2 / VITS（GPU FP16）+ HiFi-GAN（GPU）
      → 批量 batch 合并处理，排队不满 10ms 等一批再一起合成（吞吐 +80%）
      │
      ▼
CDN 分发缓存 + 播放器流式拉流
```

单路合成成本测算：1 千字文本 ≈ 10 分钟音频 ≈ 0.02 元（自建 GPU），对比商业 API（0.1~0.5 元/千字），业务量一上来自建非常划算。

### 5. 版权 & 合规防护

克隆明星/亲友声音是法律灰色地带，生产系统必须加：

- **输入音频声纹黑名单比对**（明星、公众人物 1:N 匹配，相似度 > 0.92 拒合成）
- **输出音频加水印**（用 AAC 水印算法在音频不可闻频段嵌入 24bit ID → 可溯源）
- **用户授权 + 声纹自证**（克隆自己声音需要读指定动态文本 10 秒，证明是本人授权）

## 常见坑 & 踩过的血泪

### 坑 1：长句合成到后半段音调乱飘

Transformer 自回归结构通病，长文本（> 100 字）越往后越不可控：

- **工程解法**：智能分句（按语义/标点 30~50 字一切，每段独立合成）+ 段尾段首 overlap 拼接，连接处用 WSOLA 平滑波形
- 每合成 5 段重置一次 KV Cache，防止注意力漂移

### 坑 2：声音克隆的「音色泄漏 / 情感不对」

克隆了一个「活泼女生」声音，结果合成悲伤文字时还是笑着读：

- **正确做法**：参考音频提取 speaker embedding（只存音色），不要存情感；情感用单独的 emotion embedding 注入
- 参考音频必须 **中性朗读 + 无情绪 + 无背景音**，不然把情绪也克隆进去了

### 坑 3：中英文混读 / 数字序列读出来卡顿

- 中文模型读英文单词时按字母一个个念 → 「APP」 读成「诶皮皮」→ 崩
- **解法**：双语字典 + Code-Switch 专用发音词典；CosyVoice / Fish Speech 原生支持多语言混读，中文无缝切换英文单词

### 坑 4：PPPoE（电话线路）音质下 MOS 暴跌

TTS 在 22kHz 高音质下听着不错，一到电话 PSTN 线路 8kHz ALaw/μLaw 压缩后，字都听不清了：

- **训练时做数据增强**：训练音频每次加一遍「编码成 8kHz G.729 再解码」的增强，强制模型学电话线路下也清晰的发音
- 通话 TTS 单独训一套 8kHz 采样率专用模型，不要共用 24kHz 模型降采样

## 未来方向

1. **实时对话 TTS（毫秒级）**：目前合成要 100~300ms，未来和 ASR 结合成「流式对话大模型」——用户还在说着上半句，AI 已经预测后半句并提前合成回答的前几个字，做到「真·打断式对话」，完全和真人对话体感一致。
2. **歌声 + 语音统一合成**：一句话里前半段说后半段唱、中间带笑声哭声抽泣、模拟老人小孩沙哑声——一套模型搞定所有人类声音表达。
3. **跨年龄 / 跨语种迁移**：给我你现在 30 岁声音，生成你 5 岁 / 70 岁时的声音；或用中文克隆声音后直接让它说流利英语、日语（不用重新录）。
4. **TTS 作为多模态大模型原生能力**：未来 LLM（GPT-6/Gemini 3 级别）不再需要外挂 TTS 模块——直接是文本 + 语音 + 图像三模态输出，模型内部直接理解「这段文字用悲伤的小孩子声音读」，一体化生成。

TTS 走过了 50 年的发展历程：从「像机器人说话」→「像真人说话」→「像特定真人说话，3 秒就能克隆」。下一个阶段，**它将是 AI 智能体的「声带」**——是人和 AI 物理世界交互中，除了屏幕之外最自然、最有情感温度、最不可或缺的输出通道。
