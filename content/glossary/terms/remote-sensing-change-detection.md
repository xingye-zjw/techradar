# 遥感变化检测

**遥感变化检测（Remote Sensing Change Detection）** 是对同一地理区域、不同时相获取的遥感影像（卫星、无人机、航空摄影）进行自动分析，识别出其间地表覆盖或土地利用发生变化的位置、范围和类型的技术。它是国土资源监测、城市扩张分析、森林砍伐追踪、灾害应急评估、违法建筑筛查等场景的核心技术底座。

## 变化检测问题定义

形式化表述：给定配准后的双时相影像 $I_t \in \mathbb{R}^{H \times W \times C}$（时相 $t$）和 $I_{t+\Delta t} \in \mathbb{R}^{H \times W \times C}$（时相 $t+\Delta t$），输出变化掩码 $M \in \{0,1\}^{H \times W}$，其中 $M_{hw}=1$ 表示像素 $(h,w)$ 发生了「语义上有意义」的变化。

关键难点是：**伪变化 vs 真变化的区分**

| 伪变化（需要过滤）                     | 真变化（需要检出）                |
| -------------------------------------- | --------------------------------- |
| 光照角度变化（不同成像季节/时段）      | 新增建筑物 / 建筑拆除             |
| 大气散射、云层阴影                     | 新建道路、桥梁施工                |
| 季节性植被变化（冬夏叶色差异）         | 森林砍伐、林地退化                |
| 传感器差异（Landsat 5→8 光谱响应不同） | 水域面积变化（湖泊干涸/洪水扩张） |
| 配准误差导致的像素平移                 | 山体滑坡、地震地表破坏            |
| 潮汐水位差（沿海滩涂）                 | 土地类型转换（农田→工厂）         |

## 传统方法（深度学习之前）

### 1. 像素级代数方法

- **图像差分**（Image Differencing）：$D = |I_1 - I_2|$，阈值化得到变化区。简单但对光照极敏感。
- **变化向量分析（CVA）**：每个像素视为光谱向量，计算两时相间向量模长和方向，向量方向变化对应地物类型变化。比差分更鲁棒。
- **比值法**：$D = I_1 / I_2$，对乘性光照变化有一定不变性。
- **植被指数差分**：计算 NDVI（归一化植被指数）、NDWI（水体指数）后做差分，专门对应植被/水体变化。

### 2. 变换域方法

- **主成分分析（PCA）**：将两时相影像拼接后做 PCA，取包含差分信息的次要主成分重建差异图。
- **慢特征分析（SFA）**：假设地物变化是稀疏、低频的，通过约束特征随时间变化最小化来提取不变特征。
- **MAD（多元变化检测）**：基于典型相关分析（CCA）找两时相间最大差异的线性组合，是传统方法中精度最高的一类。

传统方法局限：依赖手工设计特征，无法捕捉高阶语义变化（例如：都是混凝土，从「空地」变成「路面」肉眼难分，但语义已变）。

## 深度学习时代：基于孪生网络的像素级变化检测

### 基础架构：Siamese Encoder + Fusion + Decoder

```
     ┌─────────────┐      ┌─────────────┐
     │  时相 T1 影像 │      │  时相 T2 影像 │
     └──────┬──────┘      └──────┬──────┘
            │   Share Weights    │
            ▼                    ▼
     ┌─────────────────────────────────────┐
     │   Twin Backbone (ResNet/ViT/Swin)    │
     │   ┌────┐ ┌────┐     ┌────┐ ┌────┐   │
     │   │Conv1│ │Conv2│ ... │Conv1│ │Conv2│   │
     │   └────┘ └────┘     └────┘ └────┘   │
     └──────────┬──────────────┬───────────┘
                │  F1 多尺度   │  F2 多尺度
                └───────┬──────┘
                        ▼
            ┌────────────────────┐
            │   特征融合模块      │  ← 这里是创新主战场
            │  (Difference /     │
            │   Concat /         │
            │   Attention /      │
            │   Cross-Corr)      │
            └──────────┬─────────┘
                       ▼
     ┌──────────────────────────────────┐
     │  U-Net 风格解码器 + Skip Conn    │
     └──────────────┬───────────────────┘
                    ▼
     变化掩码 M (H×W×1 sigmoid 输出)
```

### 融合模块的四大范式

| 范式           | 代表工作                  | 思路                                                         | 优点               |
| -------------- | ------------------------- | ------------------------------------------------------------ | ------------------ |
| **差分融合**   | FC-EF、FC-Siam-Diff       | $F_{fuse} =                                                  | F_1 - F_2          | $，然后送入解码器 | 简单，参数量少 |
| **拼接融合**   | FC-Siam-Conc、STANet      | $F_{fuse} = \text{Concat}(F_1, F_2)$ + 1×1 Conv 学习通道融合 | 保留两时相各自信息 |
| **注意力融合** | DASNet、BIT、ChangeFormer | Cross-Attention：T1 的 Q 去 attend T2 的 K,V，反之亦然       | 建模长距离时空依赖 |
| **互相关融合** | SNUNet、Coal              | 在多尺度上计算 3×3 局部互相关，显式衡量邻域相似度            | 捕捉细粒度结构变化 |

### 里程碑：BIT（Bitemporal Image Transformer，2022）

首次将纯 Transformer 引入变化检测：

- 用两个独立 tokenizer 将 1/4、1/8、1/16 分辨率特征转为 N 个 token
- 用一个共享的 Transformer Encoder 处理 $[F_1; F_2]$ 拼接序列，中间插入 Cross-Token Interaction 模块
- 在 LEVIR-CD 256×256 数据集上 IoU 达到 91.5%，远超此前 CNN 的 87% 水平

### 里程碑：ChangeFormer（2022）

Swin Transformer 骨干 + 多层级 Transformer 融合解码器，是目前工业界落地最广的方案：

```python
# 使用 mmsegmentation 加载 ChangeFormer
from mmseg.apis import inference_segmentor, init_segmentor

config = "changeformer_config.py"
checkpoint = "changeformer_levir.pth"
model = init_segmentor(config, checkpoint, device="cuda:0")

img1 = "t1_satellite.tif"
img2 = "t2_satellite.tif"
result = inference_segmentor(model, [[img1, img2]])[0]
# result: H×W 数组，0 = 未变化，1 = 变化
```

## 训练工程最佳实践

### 1. 数据增强

遥感影像的有效数据增强与自然图像不同：

```python
# 遥感专属：
A.RandomRotate90(p=0.5),                     # 90° 旋转（卫星俯视无方向先验）
A.HorizontalFlip(p=0.5),
A.VerticalFlip(p=0.5),
A.OneOf([                                     # 模拟季节/天气伪变化
    A.RandomBrightnessContrast(0.2, 0.2),
    A.HueSaturationValue(hue_shift=10),
    A.Blur(blur_limit=3)
], p=0.6),
A.ShiftScaleRotate(shift_limit=0.02,         # 模拟配准误差对齐抖动
                   scale_limit=0.0,
                   rotate_limit=0, p=0.3)
```

关键点：**模拟伪变化的增强必须加，否则模型会把季节光照差当变化**。

### 2. 损失函数设计

遥感变化检测极度不平衡（变化像素通常 < 5%），不能只用 BCE：

```python
loss = (
      0.3 * BCELossWithLogits()          # 基础二分类
    + 0.5 * DiceLoss()                   # 应对类别不平衡（直接优化 IoU）
    + 0.2 * FocalLoss(gamma=2.0, alpha=0.75)  # 聚焦难分边界像素
)
```

### 3. 测试时增强（TTA）

遥感影像测试中 5× TTA 几乎无代价：

```python
flips = [identity, hflip, vflip, rot90, rot180]
for flip in flips:
    pred = model(flip(T1), flip(T2))
    avg += inverse_flip(pred)
avg /= 5
# 平均 IoU 提升 1.5-2.5 个点
```

## 语义级变化检测（Semantic CD）

像素级只判断「变 / 不变」，语义级还要回答 **「从什么变成了什么」**，输出每个像素的 (class_before, class_after)。

数据集：SECOND、Hi-UCD
模型：Symmetric Unet with dual classification heads，对每个像素输出两个 K 类概率向量 $P_t, P_{t+1}$，用：

- 交叉熵分别监督 $P_t$ 和 $P_{t+1}$
- 额外用 K×K 转移矩阵惩罚不合理的转移（如「水域 → 建筑物」在物理上几乎不可能）

## 工程落地：城市违建筛查流水线

```
┌──────────────────────────────────────────────────────────┐
│  步骤 1：数据获取                                          │
│  - Google Earth / Maxar 拉取 T0（去年）、T1（本月）影像     │
│  - 下载对应行政区划矢量作为地理 ROI 边界                    │
└───────────────────────┬──────────────────────────────────┘
                        ▼
┌──────────────────────────────────────────────────────────┐
│  步骤 2：预处理                                            │
│  - 基于 SIFT + RANSAC 做跨时相子像素级配准（RMSE < 0.5 像素）│
│  - 辐射归一化（直方图匹配，把 T1 颜色分布对齐到 T0）         │
│  - 云/云影剔除（用 Fmask 算法或专门的云检测模型）            │
└───────────────────────┬──────────────────────────────────┘
                        ▼
┌──────────────────────────────────────────────────────────┐
│  步骤 3：推理                                              │
│  - 滑窗 512×512，重叠 128 像素（保证边缘不 seam 错位）      │
│  - ChangeFormer-L 推理，Dice + BCE 后处理                  │
│  - 并行 8 GPU，处理 100km² ≈ 20 分钟                       │
└───────────────────────┬──────────────────────────────────┘
                        ▼
┌──────────────────────────────────────────────────────────┐
│  步骤 4：后处理 + 业务规则过滤                              │
│  - 连通域分析，剔除 < 25m²（约 100 像素）的噪点连通域        │
│  - 结合历年土地利用规划图：基本农田区变化 → 优先级最高       │
│  - 建筑轮廓匹配：与已有建筑矢量叠交，排除 90%+ IoU 的存量房   │
│  - 人工复核队列：只把置信度 0.4~0.8 的可疑区域推给人工       │
└──────────────────────────────────────────────────────────┘
```

这套流水线相比纯人工核查，处理效率提升 **30~50 倍**，遗漏率 < 5%。

## 未来方向

1. **时相一致性预训练**：在海量无标注多时相影像上做自监督（掩码学习 + 对比学习）预训练骨干，再迁移到小样本标注场景，解决某些区域（如非洲）标注数据极度匮乏问题。
2. **多传感器融合 CD**：融合光学影像（Sentinel-2）+ SAR 雷达（Sentinel-1，不受云遮挡）+ 高程（DEM），构建全天候变化检测系统。
3. **大模型开放词汇变化**：用 VLM（如 GPT-4V）支持自然语言驱动的变化检测——用户问「这个区域过去一年新增了多少光伏电站？」模型自动找出变化并做计数。
4. **边缘端实时 CD**：用 YOLO-CD 轻量化架构 + INT8 量化，在无人机飞控板上实时做变化检测，飞一圈直接下传变化矢量。

遥感变化检测是一个「算法精度每提升 1 个百分点，就能为政府/企业节省千万元审计/核查成本」的典型 To G/B 场景，随着卫星分辨率和重返频率的持续提升，其工程价值仍在快速放大。
