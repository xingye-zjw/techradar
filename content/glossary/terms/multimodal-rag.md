# 多模态RAG

**多模态 RAG（Multimodal Retrieval-Augmented Generation）** 是传统文本 RAG 向图像、音频、视频、3D 模型等多模态文档扩展的技术框架。它解决了传统文本 RAG 的核心盲区：企业知识库中超过 **60%** 的信息其实存储在 PDF 图表、产品示意图、培训视频、操作语音记录等非文本格式中——纯文本 RAG 对这些内容「看不见、读不懂、搜不出」。

## 为什么需要多模态 RAG？

先看一个真实的用户问题场景：

```
用户上传了一份 500 页的工业机器人维护手册 PDF（内含：
  - 200 张爆炸图、零件位置示意图
  - 30 张表格（螺栓扭矩对照表、润滑油型号参数）
  - 10 个故障排查流程图
）

然后提问：「这款 IRB-6700 机器人手腕轴密封圈怎么换？
         需要什么型号的工具？第几步要注意扭矩？」
```

纯文本 RAG 的处理方式：

1. 用 PyPDF / Unstructured 抽取文字 → **图里的步骤全丢了**
2. 做文本分块 → **表格被切成碎片，列名和数值错位**
3. 向量检索 → **搜到的全是不相关文字段落**
4. LLM 回答 → **胡说八道，缺步骤、缺工具型号**

多模态 RAG 的处理方式：

1. 文档按页分块 → 每页同时保留「页面文本」+「页面图像」
2. 用多模态编码器（CLIP / BLIP-2 / InternVL）把两种模态映射到统一嵌入空间
3. 用户提问时同时检索「相关文本块」和「相关图像块」
4. 把文本块 + 图像块一起喂给 VLM（GPT-4V、InternVL2）
5. VLM 看到爆炸图中的密封圈位置，对照文字步骤，给出**带图注引用的精准回答**

## 三大技术难点与解决方案

### 难点 1：文档图像的细粒度信息密度问题

PDF 渲染成 1024×768 的图像后，一页上的小字、标注的尺寸数字箭头 → VLM 放大 2× 也看不清，导致数字读错。

**解决方案：高分辨率切片嵌入（Hi-Res Slice Embedding）**

```
一页 A4 PDF 渲染成 300 DPI → 2480 × 3508 像素
  ↓
不做 Resize，切成 6 块（2×3 网格，每块 1240×1169，重叠 128 像素）
  ↓
每块单独用 InternVL 提取嵌入 → 存 6 个向量（关联同一页元数据）
  ↓
检索时按块命中 → 还原出是哪一页哪一区域的内容
```

这样做的代价是每页嵌入量 ×6，但图表小字号识别准确率从 55% 提升到 **93%+**。

### 难点 2：表格的结构化 vs 图像化取舍

产品参数表（列 × 行 = 型号 × 扭矩/重量/功率）纯图像喂给 VLM → 回答经常行列错位。

**解决方案：双通道表格处理**

| 通道       | 方法                                                               | 适用场景                                   |
| ---------- | ------------------------------------------------------------------ | ------------------------------------------ |
| 结构化通道 | PaddleStructure / Table Transformer 把表格结构化为 HTML / Markdown | 规整表格（有行列线、单元格完整）           |
| 图像通道   | 保留原表格渲染图 + 单独的列名 OCR                                  | 复杂层级表头、跨列合并单元格、老手册扫描件 |

检索时两个通道都做嵌入和召回，VLM 自己决定看 Markdown 还是看原图。

### 难点 3：视频/音频的时序关联检索

用户上传一段 30 分钟的机器人操作培训视频，问「拆手腕密封圈时，操作员第几步用了扭矩扳手？」——纯文本 ASR 搜索「扭矩扳手」会命中 20 次，无法对应「拆手腕」这个上下文。

**解决方案：时序滑动窗口 + 多帧联合嵌入**

```
视频切分：按 5 秒一段（25 帧），步长 1 秒，做重叠窗
每段特征：
  帧特征：中间帧 + 3 个采样帧 → 平均池化 → e_v
  音频特征：5 秒梅尔频谱 → Qwen-Audio → e_a
  字幕特征：对应 5 秒 ASR 文本 → BGE-M3 → e_t
  融合：Concat(e_v, e_a, e_t) → 小 Transformer 交叉注意力 → e_segment
检索：用户问题向量同时和 e_segment 的三个子空间做多相似度加权组合
返回：命中的 5 秒时间段 + 前后上下文扩展 15 秒 → 连同对应帧截图送入 VLM
```

## 开源生态与工程选型

### 多模态嵌入模型选型

| 模型                  | 模态   | 维度 | 中文支持   | 特点                               |
| --------------------- | ------ | ---- | ---------- | ---------------------------------- |
| **BGE-M3**            | 文本   | 1024 | ⭐⭐⭐⭐⭐ | 多粒度、双语、跨任务最强文本嵌入   |
| **CLIP ViT-L/14@336** | 图文   | 768  | ⭐⭐⭐     | OpenCLIP 中文版，稳定，速度快      |
| **InternVL2-8B**      | 图文   | 2560 | ⭐⭐⭐⭐⭐ | MMBench 接近 GPT-4V，中文 OCR 极强 |
| **Qwen-VL-Max**       | 图文   | 4096 | ⭐⭐⭐⭐⭐ | 长文档理解，动态分辨率             |
| **ImageBind**         | 六模态 | 1024 | ⭐⭐       | 图/文/音/深度/IMU/热成像统一嵌入   |

### 向量库支持

所有主流向量库（Milvus、Qdrant、Weaviate、Chroma、PGVector）都原生支持**多向量字段**：一张 Document 表同时放 `embedding_text`、`embedding_image`、`embedding_audio` 三个向量列，检索时多列相似度做加权和。

### Reranker 多模态支持

纯文本 Reranker 无法评估「图像块和问题」的相关性。选多模态版 Reranker：

- **BAAI/bge-reranker-v2-m3**：官方支持图文对打分
- **BGE-Multilingual-Multimodal-Reranker**：中英双语，图文统一打分

## 多模态 RAG 质量评估

纯文本 RAG 的指标（Context Precision/Recall、Faithfulness）都能用，但要加三组多模态专属评测：

### 1. 视觉引用准确率（Visual Citation Precision）

> 回答中引用的 I-1、I-3 图像编号，是否真的包含回答中提到的视觉信息？

人工评测 100 条问题 → 打分：引用图像真包含 = +1，张冠李戴 = 0，引用完全无关图像 = -1。工业落地要求 ≥ 0.85。

### 2. 图表数值正确率（Chart/Tabular Accuracy）

构造 50 道纯图表问答题（只看图像中的表/图才能答出）：

- 正确回答 = 1
- 数值错误 ±5% 以内 = 0.5
- 列行错位或完全错误 = 0

要求正确率 ≥ 80%（纯文本 RAG 通常 < 30%）。

### 3. 跨模态召回率（Cross-Modal Recall@K）

给定 100 个纯图像答案的问题，检索 Top-K 是否能命中正确图像块。用 Recall@10 衡量，要求 ≥ 90%。

## 未来方向

1. **端到端多模态 RAG 大模型**：现在是「检索 + 生成」两阶段。新架构（如 Gemini 原生长上下文 + 多模态检索插件）会把检索器做进模型内部，用户无感。
2. **3D 模型 / CAD 图纸支持**：制造业下一步需求是搜 3D STP 装配体模型——把 B-Rep 几何、PMI 标注、物料清单统一嵌入。
3. **交互式多模态检索（Conversational Multimodal RAG）**：用户问一句后 VLM 反过来说「你说的密封圈我拿不准，是不是截图这张图里红圈标的零件？」→ 用户确认 → 再做精准检索和回答。
4. **合规与溯源增强**：每个引用图像块旁边挂一个「溯源二维码」，点开跳转到原 PDF 原页原位置，满足医疗/航空/法务的「AI 回答必须可审计追溯」要求。

多模态 RAG 是让 LLM 真正从「会聊天的文字处理机」进化成「能看懂企业全部信息资产的知识工作者」的关键一跳。文本 RAG 解决了「知道」的问题，多模态 RAG 解决了「看见」的问题——两者结合，才是 RAG 系统的完整形态。
