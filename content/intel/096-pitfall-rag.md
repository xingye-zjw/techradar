## RAG 检索到了但回答不对 / 幻觉严重

// 快速修复

换中文 embedding + rerank + 明确引用约束的 Prompt

// 现象表现

- × 向量搜索返回了相关文档但LLM回答与文档不符
- × 答案引用了训练知识而非检索文档
- × 中英文混排检索结果差

// 排查步骤

- 01 选用BGE-M3等中文表现强的embedding
- 02 chunk size取200-400 token设10-20% overlap
- 03 用reranker二次排序
- 04 Prompt明确约束仅基于给定上下文回答
- 05 HyDE模式改写query
- 06 检查向量数据库chunk是否正确

## 向量数据库检索结果与预期不匹配

// 快速修复

检查 embedding 模型 + 调整 chunk_size + 验证向量维度

// 现象表现

- × 语义相关的内容检索不到
- × 检索结果高度相似但不是想要的
- × 高频内容主导检索

// 排查步骤

- 01 验证embedding模型在目标任务上的表现
- 02 调整chunk size避免信息碎片化
- 03 检查向量维度是否一致
- 04 考虑使用混合搜索(向量+关键词)

## RAG 系统延迟高/响应慢

// 快速修复

异步加载 + 向量索引优化 + 缓存检索结果

// 现象表现

- × 用户提问后等待时间长
- × 检索阶段耗时>生成阶段
- × 并发请求时延迟剧增

// 排查步骤

- 01 使用异步加载embedding模型
- 02 向量数据库加索引(HNSW/IVF)
- 03 缓存热门query结果
- 04 使用批量推理

## 知识库内容切分不合理导致关键信息丢失

// 快速修复

按语义段落切分 + 设置 overlap + 特殊文档特殊处理

// 现象表现

- × 检索到的片段不完整无法回答
- × 关键信息被切断在两个chunk中
- × 答案需要跨段落推理

// 排查步骤

- 01 按自然段落/句子边界切分
- 02 设置10-20% overlap
- 03 对表格/代码等特殊内容用专用切分器
- 04 避免单段落过长
