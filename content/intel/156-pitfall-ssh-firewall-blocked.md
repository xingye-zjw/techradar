## 为什么你要学它

这是开发中非常容易踩的一个坑：**SSH 端口被防火墙拦截 / 服务器无法访问**。

SSH 无法连接远程服务器，即使服务正常运行。通常是云服务器安全组配置错误或网络防火墙拦截所致。

如果你正在遇到类似问题，或者想提前预防，这篇卡片会帮你快速定位问题、找到解决方案，并学会从根源上避免它。

## 一句话概览（快速版）

> **快速修复：检查安全组规则 → 改端口 443 → 开启密钥登录 + fail2ban**

核心要点：

- **现象**：ssh: connect to host x.x.x.x port 22: Connection timed out
- **根因**：云服务器安全组未开放 SSH 端口、家用宽带封禁 22 端口、或防火墙规则配置错误。也可能是 ISP 对非标准端口的封锁。
- **解决**：按照下方 6 步标准流程排查

## 核心拆解

### 🔑 典型症状

- × ssh: connect to host x.x.x.x port 22: Connection timed out
- × 云服务器安全组已开端口但仍无法连接
- × 本地端口 22 被 ISP 封锁（家用宽带）

### 🔑 根本原因

云服务器安全组未开放 SSH 端口、家用宽带封禁 22 端口、或防火墙规则配置错误。也可能是 ISP 对非标准端口的封锁。

## 完整排查方案

按照以下步骤逐一排查，通常能在几分钟内定位并解决问题：

1.  云服务器检查安全组：确认入方向规则包含 SSH（22 端口或自定义端口）
2.  修改 SSH 端口为 443（绕过部分防火墙封锁）：Port 443
3.  家用宽带无法直接访问：使用内网穿透工具（frp / nps）或 cloudflare tunnel
4.  使用密钥登录替代密码，减少被暴力破解的风险
5.  安装 fail2ban 自动封禁多次登录失败的 IP：sudo apt install fail2ban
6.  长期解决方案：使用 Tailscale 或 WireGuard 建立 VPN 隧道

### 快速修复（救急用）

如果你现在就卡在这里，先试试这个：

> 检查安全组规则 → 改端口 443 → 开启密钥登录 + fail2ban

## 预防措施

知道怎么解决还不够，更重要的是**下次别再踩**：

- 云服务器部署时，第一件事就是配置安全组，开放 SSH 端口（22 或自定义端口）
- 使用密钥登录替代密码，并禁用 root 远程登录，减少暴力破解风险
- 安装 fail2ban 自动封禁多次登录失败的 IP，增强服务器安全性
- 家用宽带用户提前配置内网穿透（frp / Tailscale），避免临时无法连接

## 常见误区

1. **只看表面现象，不深挖根因** — 问题暂时解决了，但过段时间又出现
2. **跳过排查步骤直接试方案** — 容易引入新的问题
3. **不做预防措施** — 同一个坑踩好几次

## 推荐学习顺序

1. 先看「典型症状」确认你遇到的是不是这个问题
2. 再看「快速修复」试试能不能立刻解决
3. 如果不行，按照「完整排查方案」一步步来
4. 最后一定要看「预防措施」，避免下次再踩
