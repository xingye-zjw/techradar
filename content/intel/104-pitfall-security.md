## Prompt 注入攻击导致系统 prompt 泄露

// 快速修复

输入过滤 + 限制输出格式 + 不在 prompt 中包含敏感信息

// 现象表现

- × 用户输入后系统 prompt 被暴露在输出中
- × 攻击者通过特殊输入（如角色扮演、指令覆盖）绕过限制
- × 模型执行非预期操作或返回敏感配置信息

// 排查步骤

- 01 使用边界输入测试，验证是否存在 prompt 泄露风险
- 02 对用户输入进行严格的过滤和清洗，拒绝可疑指令模式
- 03 在沙箱环境中运行推理，检查输出是否包含 prompt 内容
- 04 确保敏感信息不直接写在 prompt 中，采用后端配置注入方式

## RAG 系统检索到恶意文档导致生成有害内容

// 快速修复

检索结果审核 + 输出内容过滤 + 监控检索来源

// 现象表现

- × 生成内容包含明显错误、有害或偏见信息
- × 检索到非预期的文档来源，内容与查询无关
- × 返回的答案引用了被污染的文档库内容

// 排查步骤

- 01 对检索结果做相关性过滤和来源验证，丢弃低质量结果
- 02 对生成内容进行安全审核，检测有害信息
- 03 记录检索来源日志，便于溯源和问题排查
- 04 定期清理和审核文档库，移除恶意或过期内容

## 模型 API 密钥泄露导致被恶意调用

// 快速修复

使用环境变量管理密钥 + 定期轮换 + API 访问限制

// 现象表现

- × API 调用量异常增高，超出正常使用范围
- × 收到巨额账单，发现密钥被大范围滥用
- × 模型被用于非法目的，产生法律和合规风险

// 排查步骤

- 01 检查代码仓库中是否存在硬编码的 API 密钥
- 02 立即启用密钥管理服务（如 AWS Secrets Manager、Azure Key Vault）
- 03 为 API 密钥设置调用配额和 IP 白名单限制
- 04 立即轮换泄露的密钥，并检查历史调用日志

## 对抗样本攻击导致模型误分类

// 快速修复

对抗训练 + 输入预处理 + 模型蒸馏

// 现象表现

- × 添加微小扰动后模型完全误分类，输出结果与预期相反
- × 攻击者制作特定图案或噪声绕过目标检测/分类系统
- × 在实际部署中模型表现显著下降，攻击成功率上升

// 排查步骤

- 01 测试模型对已知对抗样本的鲁棒性，评估攻击成功率
- 02 使用对抗训练（Adversarial Training）增强模型防御能力
- 03 在输入端加入去噪或预处理操作（如 JPEG 压缩、随机裁剪）
- 04 使用模型蒸馏（Distillation）将鲁棒性知识迁移到轻量模型
