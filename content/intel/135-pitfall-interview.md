## 简历堆砌技术栈但没有深度

// 快速修复

聚焦核心技能 + 量化成果 + STAR法则

// 现象表现

- × 简历过不了初筛，HR一眼扫过就丢进回收站
- × 面试官一问技术细节就露馅，支支吾吾说不清楚
- × 简历上写了十几项"精通"，真正深入问时全是"了解"

// 排查步骤

- 01 简历只写真正熟练的技术，把"精通/熟悉/了解"分清楚，宁缺毋滥
- 02 每个项目用STAR法则描述：情境(Situation)、任务(Task)、行动(Action)、结果(Result)
- 03 用数据量化成果，比如"性能提升50%""用户增长3倍""节省运维成本20万"

## 刷题但不会做变形题

// 快速修复

理解思路而非背答案 + 归类题型 + 举一反三

// 现象表现

- × 做过的原题能秒解，稍微变一下题干就懵了
- × medium题靠记忆默写，hard题全跪根本没思路
- × 面试时面试官改个条件就慌了，不知道从哪下手

// 排查步骤

- 01 每道题做完后总结套路和适用场景，问自己"这道题的本质是什么"
- 02 按题型归类刷题（动态规划/滑动窗口/回溯/二分等），而非按题号顺序刷
- 03 做完原题后主动做变形题检验理解，比如改条件、改数据规模、加follow up

## 项目经历讲不清楚

// 快速修复

STAR法则 + 讲清挑战和方案 + 突出个人贡献

// 现象表现

- × 介绍项目像流水账，从需求到上线平铺直叙，面试官听得打瞌睡
- × 面试官追问细节答不上，"这个好像是别人做的""我不太清楚当时为什么这么设计"
- × 讲了半天不知道你到底做了什么，感觉你就是个打酱油的

// 排查步骤

- 01 用STAR法则准备每个项目的30秒电梯 pitch 和3分钟详细版
- 02 讲清楚项目中最大的难点/挑战是什么，你提出了什么方案，为什么选这个方案
- 03 量化你做的贡献，明确区分团队成果和个人产出，用"我主导了""我优化了"开头

## 谈薪资时报价过低/过高

// 快速修复

调研市场行情 + 给出区间而非固定数 + 锚定策略

// 现象表现

- × 报价低了入职就后悔，看着同事的工资心里不平衡
- × 报价高了直接被pass，连还价的机会都没有
- × 不知道自己值多少钱，全靠拍脑袋和面试官瞎聊

// 排查步骤

- 01 脉脉/levels.fyi/牛客网/OfferShow查同级别同城市同行业薪资范围
- 02 给区间而非固定值，比如"期望在25-30K之间"，下限是你能接受的最低值
- 03 基于上一份薪资涨幅30%左右作为基准，结合市场行情上下浮动，不要虚报也不要贱卖
