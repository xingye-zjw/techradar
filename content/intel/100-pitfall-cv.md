## YOLO 训练时数据标注格式错误

// 快速修复

确认标注格式：YOLO 使用 cx cy w h 归一化坐标（值域 0~1）；VOC 使用 xmin ymin xmax ymax 像素坐标。不同框架不可混用，训练前务必与代码中的解析逻辑对齐。

// 现象表现

- × 检测框位置明显偏移，预测框与目标不重合
- × 模型不收敛，loss 持续波动或迟迟不下降
- × 训练 loss 正常下降，但预测结果全为背景或全类别置信度极低

// 排查步骤

- 01 随机抽取标注文件，检查每行是否为「类别ID x_center y_center width height」格式，坐标值是否在 [0, 1] 区间内
- 02 使用可视化工具（如 CVAT、LabelImg 或 Python + matplotlib）加载标注Overlay到原图，确认标注框与实际目标严格对齐
- 03 检查类别 ID 是否从 0 连续编号，无遗漏、无重复，类别名与训练配置完全一致
- 04 确认数据加载代码中的标注解析逻辑与标注文件格式匹配（如 albumentations 的 bbox_format 参数）

## 图像增强导致标注错位

// 快速修复

使用 albumentations 库的 bbox 参数进行增强，确保图像与标注框同步变换。不要手动对图像做增强后再单独处理标注坐标，两者的变换必须原子化执行。

// 现象表现

- × 数据增强后模型性能显著下降
- × 检测框与目标物体严重分离，框住空白区域
- × 训练集评估指标正常，验证集指标异常偏低

// 排查步骤

- 01 随机抽取若干增强后的样本，可视化增强后图像与标注框叠加效果，确认两者同步变换
- 02 检查增强代码是否使用了 albumentations 的 BboxParams，确保 format、label_fields 等参数正确配置
- 03 确认增强 pipeline 中所有变换（翻转、旋转、裁剪、缩放等）都同步更新了 bbox 坐标，而非只变换图像
- 04 对比手动增强（先增强图像再用公式计算新坐标）与 albumentations 增强的结果差异，优先使用后者

## 预训练模型在自定义数据上效果差

// 快速修复

冻结骨干网络（backbone）仅微调检测/分割头部，同时将数据归一化统计量对齐 ImageNet（mean=[0.485, 0.456, 0.406]，std=[0.229, 0.224, 0.225]）。

// 现象表现

- × 迁移学习后准确率低于预期，模型泛化能力差
- × 模型输出偏向 ImageNet 预训练类别，自定义类别区分度低
- × 微调后模型在自有数据上过拟合或完全失去预测能力

// 排查步骤

- 01 检查自有数据分布是否与预训练数据集（ImageNet/COCO）相似，若差异过大考虑从头训练或选择领域更接近的预训练模型
- 02 确认预处理方式与预训练时一致：输入尺寸、归一化 mean/std、数据增强策略是否对齐
- 03 微调时采用较低学习率（建议 1e-4 ~ 1e-5），并使用余弦退火或阶梯下降调度器
- 04 若自有数据集较小，可先冻结 backbone 仅微调头部，避免过拟合；若数据量充足，可逐层解冻（layer-wise finetune）

## 小目标检测效果差

// 快速修复

降低特征图下采样倍数（如从 32 倍降至 16 倍或 8 倍），使用 FPN/PAFPN 等多尺度特征融合模块，并将输入分辨率从 640×640 提升至 1024×1024 或更高。

// 现象表现

- × 小目标漏检率显著高于中大目标
- × 小目标检测框极小，置信度偏低甚至低于阈值被过滤
- × 模型在常规目标上表现良好，但小目标召回率极低

// 排查步骤

- 01 检查输入图像分辨率是否足够，若特征图过小（如 20×20），小目标信息已在下采样中丢失
- 02 检查模型特征金字塔结构，确认是否存在 FPN/PAFPN 等多尺度融合，浅层特征图是否参与预测
- 03 核实预置锚框（anchor）尺寸是否覆盖小目标范围，可通过 K-Means 聚类自有数据集中的目标尺度来重新生成锚框
- 04 在推理阶段使用多尺度测试（TTA，Test Time Augmentation），或设置更小的置信度阈值和 NMS IoU 阈值避免小目标被抑制
