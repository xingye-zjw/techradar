## 推荐系统马太效应导致头部内容越推越多

// 快速修复

加入多样性调控 + 探索新内容 + 平衡曝光

// 现象表现

- × 推荐列表高度同质化，用户刷来刷去都是相似内容
- × 长尾内容几乎不被推荐，优质新内容难以出头
- × 用户反馈集中在少数热门内容，点击率分布极度不均
- × 内容创作者积极性下降，平台生态逐渐萎缩

// 排查步骤

- 01 分析推荐结果的覆盖率和多样性，计算推荐列表中不同内容类别的占比、头部内容曝光占比、长尾内容曝光占比
- 02 加入MMR（Maximal Marginal Relevance）或diversity re-rank重排策略，在相关性基础上增加多样性约束
- 03 引入探索与利用（Exploration & Exploitation）机制，如UCB、Thompson Sampling等算法，给新内容一定的探索流量
- 04 建立内容曝光的公平性调控，设置单条内容曝光上限，强制打散同类内容
- 05 监控多样性指标（如品类覆盖率、作者覆盖率、信息熵），与核心业务指标一起纳入评估体系

## 线上线下指标不一致

// 快速修复

线上A/B测试 + 离线评估模拟线上 + 关注核心业务指标

// 现象表现

- × 离线NDCG、AUC等排序指标很高，但线上点击率、转化率不升反降
- × 模型在测试集上效果显著提升，但上线后实际业务指标没有变化甚至下跌
- × 离线评估和线上效果的相关性很低，离线指标无法指导线上迭代
- × 不同模型离线指标差距很大，但线上A/B测试无显著差异

// 排查步骤

- 01 检查离线数据是否有偏，确认训练数据是否存在位置偏置、曝光偏置、选择偏置等问题
- 02 严格做线上A/B测试验证，以线上核心业务指标（点击率、停留时长、留存率、转化率等）为准
- 03 使用全链路指标而非单一排序指标，关注用户完整生命周期的体验和转化
- 04 构建离线评估的线上模拟环境，使用前一天的数据训练、后一天的数据验证，尽量贴近线上时序
- 05 引入逆倾向得分（IPS）等方法校正离线评估中的偏置，提高离线评估的可信度
- 06 建立离线指标与线上指标的相关性追踪，定期校验离线评估的有效性

## 冷启动问题处理不当导致新用户体验差

// 快速修复

热门推荐 + 用户画像快速构建 + 主动探索

// 现象表现

- × 新用户推荐全是泛热门内容，缺乏个性化，用户觉得"不准"
- × 新用户次日留存率、7日留存率显著低于老用户
- × 冷启动期（前3-7天）推荐效果差，用户流失严重
- × 新用户互动率低，行为数据积累慢，形成恶性循环

// 排查步骤

- 01 检查新用户推荐策略，确认是否有专门的冷启动策略，还是直接复用老用户的推荐模型
- 02 利用注册信息（性别、年龄、地域、兴趣标签等）构建用户初步画像，做粗粒度个性化
- 03 主动询问用户偏好，通过引导式选择（如"选择你感兴趣的内容分类"）快速收集兴趣信号
- 04 设计热门推荐的分层策略，从泛热门→品类热门→个性化热门逐步过渡
- 05 引入主动探索机制，给新用户展示多样性更高的内容，快速收集用户反馈
- 06 利用社交关系、设备信息、安装的App等第三方数据辅助构建画像
- 07 建立新用户漏斗分析，追踪从注册到首次互动到留存的转化路径，定位流失环节

## 特征穿越导致线上效果急剧下降

// 快速修复

严格时间切分 + 特征生成与训练时间对齐 + 特征血缘追踪

// 现象表现

- × 离线评估指标极高（如AUC>0.95），但上线后效果急剧下降甚至不如随机
- × 模型在训练集和测试集上表现都很好，但一到线上就"崩"
- × 特征值在预测时实际上不可得，但训练时使用了未来信息
- × 模型对某些特征依赖度过高，但这些特征在线上无法实时获取

// 排查步骤

- 01 检查特征是否用了未来信息，重点排查统计类特征（如用户历史点击率、内容热度）的时间窗口是否正确
- 02 严格按时间窗口划分训练/验证/测试集，禁止随机划分，确保验证集和测试集的时间都晚于训练集
- 03 特征工程要模拟线上环境，训练时能拿到的特征必须与线上预测时完全一致
- 04 建立特征血缘追踪系统，记录每个特征的生成逻辑、时间依赖、数据源
- 05 做特征重要性分析，对TOP重要特征逐一排查是否存在穿越问题
- 06 上线前做"时光倒流"验证，用历史某一天的模型特征和当时的数据做模拟预测，验证效果
- 07 建立特征监控体系，线上监控特征分布、缺失率、覆盖率，发现异常及时告警
