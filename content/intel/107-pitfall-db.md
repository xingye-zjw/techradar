# 数据库避坑指南

## SQL 查询全表扫描导致数据库卡死

// 快速修复

添加合适索引 + 优化查询语句 + 使用 EXPLAIN 分析

// 现象表现

- × 简单查询耗时几十秒
- × 数据库 CPU 100%
- × 其他查询也变慢

// 排查步骤

- 01 使用 EXPLAIN 分析查询计划，定位全表扫描
- 02 检查 WHERE 条件字段是否有索引
- 03 避免 SELECT *，只查询需要的字段
- 04 对高频查询字段建立合适索引
- 05 优化 SQL 语句结构

## 数据库连接池耗尽导致应用无响应

// 快速修复

配置合适的连接池大小 + 设置连接超时 + 实现连接池监控

// 现象表现

- × 应用卡死无响应
- × 数据库报错 too many connections
- × 间歇性服务中断

// 排查步骤

- 01 检查连接池配置，确认最大连接数设置合理
- 02 确认连接是否正确释放，确保 finally 块释放连接
- 03 监控连接池使用率，设置告警
- 04 检查是否有连接泄漏（未关闭的连接）
- 05 设置合理的连接超时时间

## 事务未提交导致锁等待超时

// 快速修复

确保事务尽快提交 + 添加合理的超时设置 + 避免长事务

// 现象表现

- × 查询卡住不动
- × 出现 lock wait timeout 错误
- × 部分操作失败

// 排查步骤

- 01 检查未提交的事务，定位长时间运行的事务
- 02 使用 SHOW PROCESSLIST 查看锁状态
- 03 确保 finally 块正确释放连接和提交事务
- 04 设置合理的 lock_wait_timeout 参数
- 05 拆分长事务为多个短事务

## 数据库主从延迟导致读写分离失效

// 快速修复

读写分离 + 延迟监控 + 强制读主库解决

// 现象表现

- × 写入后立即读取拿不到数据
- × 从库数据比主库慢几秒

// 排查步骤

- 01 监控主从同步延迟，关注 Seconds_Behind_Master
- 02 设置合理的复制相关配置（如 sync_binlog）
- 03 对一致性要求高的读操作强制走主库
- 04 检查从库是否有大事务阻塞复制
- 05 优化网络延迟或升级硬件
