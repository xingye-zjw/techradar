## 奖励函数设计不当导致Agent走捷径

// 快速修复

奖励函数细粒度化 + 加入中间奖励 + 用负奖励惩罚不良行为

// 现象表现

- × Agent找到了比人类更高效的得分方式但不是预期行为
- × Agent卡在某个状态重复动作
- × Agent学习到的策略与目标背道而驰

// 排查步骤

- 01 分析reward shaping是否合理，检查是否存在奖励漏洞
- 02 加入中间奖励引导学习，逐步逼近最终目标
- 03 加入负奖励惩罚错误行为，抑制不良策略
- 04 用稀疏奖励替代密集奖励，减少奖励塑形带来的副作用

## 探索与利用失衡导致训练不稳定

// 快速修复

ε衰减 + 熵正则化 + 多随机种子

// 现象表现

- × 训练曲线波动极大，Reward上下剧烈震荡
- × 在某个性能平台上长期停滞，无法进一步提升
- × 不同seed结果差异巨大，实验可复现性差

// 排查步骤

- 01 检查ε-greedy的ε衰减设置，确保从充分探索逐步过渡到利用
- 02 加入熵正则化鼓励探索，防止策略过早收敛到次优解
- 03 跑多个seed取平均，减少随机性对结果的影响
- 04 调整学习率与探索率的衰减节奏，确保二者匹配

## 策略梯度方差过大导致训练崩溃

// 快速修复

加入Baseline + GAE优势估计 + 梯度裁剪

// 现象表现

- × 训练几步后reward直接掉到负无穷
- × 策略梯度方差极大，参数更新剧烈
- × 网络输出突然变极端，动作概率分布塌陷

// 排查步骤

- 01 使用优势函数替代原始reward，减少方差
- 02 加入基线(baseline)减少方差，常用状态价值函数作为基线
- 03 使用GAE广义优势估计，平衡偏差与方差
- 04 加入梯度裁剪，防止梯度爆炸导致训练崩溃

## 经验回放缓冲区样本质量差

// 快速修复

优先级回放 + 限制buffer大小 + 定期清理旧样本

// 现象表现

- × off-policy算法训练效率低，收敛速度慢
- × 学到的策略不稳定，性能波动大
- × 样本分布与当前策略差异大，产生分布偏移问题

// 排查步骤

- 01 使用优先级经验回放(PER)，提高重要样本的采样概率
- 02 控制replay buffer大小，避免过旧的样本主导训练
- 03 加入更近期的样本，缩小样本分布与当前策略的差距
- 04 定期清理旧样本，保持缓冲区样本的时效性
