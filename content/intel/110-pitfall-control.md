## PID 积分饱和导致系统超调过大

// 快速修复

实现积分限幅 + 抗积分饱和算法（Anti-windup）+ 微分先行结构 + 设定值低通滤波

// 现象表现

- × 系统响应超调量过大，超调率超过 20%
- × 振荡次数明显增多，收敛时间变长
- × 设定值突变时控制输出剧烈变化，执行机构动作幅度过大
- × 积分时间越长，系统响应越容易发散

// 排查步骤

- 01 检查积分项是否设置上限幅（通常设为输出范围的 80-100%）
- 02 验证是否实现了 Anti-windup 机制，推荐使用条件积分法或回退法
- 03 检查微分项是否采用微分先行结构（对反馈值滤波而非对设定值）
- 04 确认设定值变化时是否经过斜率限制或低通滤波处理
- 05 检查控制器输出是否进入执行机构的饱和区

## 离散控制系统采样频率设置不当

// 快速修复

采样频率设为系统闭环带宽的 10-20 倍 + 使用双线性变换（Tustin）离散化 + 输入端抗混叠滤波

// 现象表现

- × 系统响应与连续系统仿真结果差异显著
- × 出现不应该有的振荡或震颤
- × 奈奎斯特频率附近增益异常放大或衰减
- × 设定值跟踪出现相位滞后或超调

// 排查步骤

- 01 确认采样频率满足奈奎斯特准则：fs > 2 × 闭环带宽
- 02 检查离散化方法是否匹配系统特性（阶跃响应用双线性变换更精确）
- 03 验证离散控制器系数是否正确实现（定点化时注意精度损失）
- 04 在传感器输入端添加抗混叠低通滤波器，截止频率设为 fs/2 的 40-60%
- 05 若系统带宽无法降低，考虑提高采样频率或降低闭环带宽要求

## 状态空间控制器极点位置选择不当

// 快速修复

使用 LQR 设计方法确定最优增益矩阵 + 验证所有极点位于单位圆内（离散系统）+ 观察状态收敛速度与控制能量权衡

// 现象表现

- × 系统响应过慢，状态变量需要很长时间才能收敛
- × 系统响应过快但控制输入过大，执行机构饱和
- × 状态变量出现振荡，尤其是高阶系统
- × 闭环系统不稳定，状态发散

// 排查步骤

- 01 使用 MATLAB/Simulink 或 Python control library 设计 LQR 控制器
- 02 检查 Q、R 权重矩阵的物理意义：Q 越大响应越快，R 越大控制能量越小
- 03 验证闭环极点位置，确保离散系统所有极点在单位圆内
- 04 仿真检查状态轨迹和控制输入是否满足执行机构约束
- 05 进行灵敏度分析，验证模型参数摄动时系统的鲁棒性

## 传感器噪声导致微分项失效

// 快速修复

添加低通滤波（截止频率 5-10 倍带宽） + 微分先行结构 + 噪声预处理 + 适当增大微分时间常数

// 现象表现

- × 控制输出剧烈抖动，执行机构频繁小幅动作
- × 执行机构磨损加剧，寿命缩短
- × 系统出现高频振荡，噪声频谱明显
- × 微分项效果不明显，抑制超调能力下降

// 排查步骤

- 01 对测量信号进行低通滤波处理，推荐使用二阶巴特沃斯滤波器
- 02 检查微分项是否实现了带滤波的微分：Td*s/(1+Td*s/N)
- 03 适当增大微分时间常数 Td，牺牲部分微分效果换取稳定性
- 04 验证传感器信号质量，检查信噪比和接地情况
- 05 考虑使用状态观测器估计真实状态，而非直接使用噪声测量值

## 修复后附加：最小一键诊断命令

```bash
/* 嵌入式最小诊断：时钟+GPIO+UART 三件套寄存器自检（STM32 参考，可直接替换为自己的 MCU） */
#include <stdint.h>
volatile uint32_t *RCC_AHB1 = (volatile uint32_t *)0x40023830UL;
volatile uint32_t *GPIOD_MODER = (volatile uint32_t *)0x40020C00UL;
volatile uint32_t *GPIOD_ODR   = (volatile uint32_t *)0x40020C14UL;
int mcu_self_test(void) {
  *RCC_AHB1 |= (1UL << 3);          /* GPIOD 时钟使能 */
  *GPIOD_MODER &= ~(3UL << (12*2)); /* PD12 -> 输出 */
  *GPIOD_MODER |=  (1UL << (12*2));
  *GPIOD_ODR  |=  (1UL << 12);      /* 置高 PD12 LED */
  return (*GPIOD_ODR >> 12) & 1U;   /* 回读应为 1 */
}
```
