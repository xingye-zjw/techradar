## 二分查找边界条件处理错误

// 快速修复

采用左闭右闭区间统一写法：`mid = l + (r - l) / 2`，循环条件为 `l <= r`，找到返回 mid，否则根据大小关系移动 `l = mid + 1` 或 `r = mid - 1`。

// 现象表现

- × 死循环，程序运行不结束
- × 目标元素明明存在却找不到（漏找）
- × 边界值（数组第一个或最后一个元素）找不到
- × 数组为空时访问越界

// 排查步骤

- 01 明确区间定义：是左闭右闭 `[l, r]` 还是左闭右开 `[l, r)`，全程保持一致
- 02 检查 mid 计算方式：用 `l + (r - l) / 2` 替代 `(l + r) / 2` 防止整数溢出
- 03 核对循环条件：左闭右闭对应 `l <= r`，左闭右开对应 `l < r`
- 04 验证边界移动：左闭右闭时未找到要 `l = mid + 1` 或 `r = mid - 1`，不能保留 mid

## 递归爆栈/栈溢出

// 快速修复

优先改写为迭代（用显式栈模拟）；无法改写的尝试尾递归优化；存在重复子问题时加记忆化缓存；Python 环境下可临时调大 `sys.setrecursionlimit`。

// 现象表现

- × 大数据量时抛出 `StackOverflowError` 或 `RecursionError`
- × 递归深度超过 Python 默认限制（默认约 1000 层）
- × 程序运行到一定规模后突然崩溃
- × 相同输入反复计算导致性能极差，间接加深递归层级

// 排查步骤

- 01 分析递归深度：估算最坏情况下递归调用栈层数，确认是否超出语言/系统限制
- 02 尝试迭代改写：用栈数据结构手动维护状态，将递归转为循环
- 03 检查重复计算：画出递归树，若存在大量重复子问题则加入记忆化（memoization）
- 04 Python 环境可调大递归深度：`sys.setrecursionlimit(100000)`，但这只是临时方案
- 05 尾递归优化：确保递归调用是函数最后一步操作（依赖语言/编译器支持）

## 哈希表键的可变类型陷阱

// 快速修复

使用不可变类型（字符串、数字、元组）作为字典键；自定义类作为键时，必须同时正确实现 `__hash__` 和 `__eq__` 方法，且保证哈希值在对象生命周期内不变。

// 现象表现

- × 字典中明明插入了键，后来却找不到（KeyError）
- × 修改对象属性后，哈希值变化，无法从字典/集合中取回
- × 两个逻辑相等的对象被当作不同的键存入字典
- × 集合（set）中出现"重复"元素

// 排查步骤

- 01 确认键的类型：检查是否使用了列表、字典等可变类型作为键（Python 会直接报错）
- 02 追踪对象修改：对象放入哈希表后，是否修改了影响哈希值的属性
- 03 自定义类检查：是否同时实现了 `__hash__` 和 `__eq__`，二者必须一致（a==b 则 hash(a)==hash(b)）
- 04 验证哈希不变性：对象作为键使用期间，参与哈希计算的字段是否保持不变

## 动态规划状态转移方程错误

// 快速修复

先明确定义 `dp[i]` 或 `dp[i][j]` 表示的具体含义，再用小规模例子手推验证，检查初始条件是否正确，最后列举所有可能的转移来源确保无遗漏。

// 现象表现

- × 动态规划计算结果与预期不符（偏大或偏小）
- × 边界条件错误导致结果偏差或数组越界
- × 转移方程漏掉某些情况，答案不全
- × 空间优化后状态覆盖出错，结果逻辑混乱

// 排查步骤

- 01 明确状态定义：写清楚 `dp[i]` 或 `dp[i][j]` 到底表示什么，定义模糊是万恶之源
- 02 手算小规模：用 n=1、n=2、n=3 等简单情况手动推导，验证方程是否正确
- 03 检查初始条件：`dp[0]`、`dp[1]` 等 base case 是否赋值正确，是否有遗漏
- 04 列举转移来源：思考 `dp[i]` 可以从哪些前面的状态转移而来，是否全部考虑到
- 05 空间优化核对：滚动数组优化时注意覆盖顺序，确保用到的旧值未被提前覆盖

## 修复后附加：最小一键诊断命令

```bash
# CS 基础最小自检：大 O + 算法复杂度自测 3 秒内
python - <<'PY'
from time import perf_counter
N = 100_000
arr = list(range(N)); s = 0
t0 = perf_counter()
for x in arr: s += x            # O(n)
t1 = perf_counter()
arr_sorted = sorted(arr)        # O(n log n)
t2 = perf_counter()
print(f'sum O(n)   = {(t1-t0)*1000:6.2f} ms  sum={s}')
print(f'sort       = {(t2-t1)*1000:6.2f} ms  first={arr_sorted[0]} last={arr_sorted[-1]}')
PY
```
