## 随机打乱时序数据导致数据泄露

// 快速修复

严格按时间顺序划分训练集与测试集，采用滚动窗口交叉验证（Rolling Window CV），数据加载与训练过程中禁止 shuffle。

// 现象表现

- × 模型离线评估指标（如准确率、R²）极高，接近完美
- × 模型上线后实际预测效果极差，与离线评估严重脱节
- × 训练集与测试集存在时间重叠或数据来自同一时间段
- × 特征重要性显示"未来信息"特征权重异常高

// 排查步骤

- 01 检查数据划分代码，确认是否使用了 `train_test_split` 等随机划分函数且未禁用 shuffle
- 02 打印训练集与测试集的时间范围，验证训练数据时间是否严格早于测试数据
- 03 改用时间序列交叉验证（如 TimeSeriesSplit、滚动窗口验证）重新评估模型
- 04 检查特征工程中是否引入了未来信息（如使用了全量数据的统计量）

## 过度差分导致ARIMA模型失效

// 快速修复

差分阶数 d 一般不超过 2，通过 ADF 检验判断平稳性，结合 ACF/PACF 图辅助确定 AR(p) 和 MA(q) 阶数。

// 现象表现

- × 差分后的序列变成白噪声，无任何自相关性
- × 模型预测置信区间极大，几乎失去预测意义
- × 预测结果呈现随机游走特征，与历史趋势脱节
- × 模型残差不满足白噪声假设，诊断检验不通过

// 排查步骤

- 01 对原始序列和差分序列分别做 ADF 检验，判断是否真正达到平稳
- 02 检查差分阶数是否超过 2，多数经济/金融数据 d=1 即足够
- 03 绘制 ACF 和 PACF 图，观察差分后序列是否仍有显著截尾/拖尾特征
- 04 对比 d=0、d=1、d=2 三种情况下的模型 AIC/BIC 值，选择最优阶数

## LSTM处理长序列时梯度消失

// 快速修复

使用 TCN（时序卷积网络）或 Transformer 替代 LSTM，对超长序列进行分段处理，引入注意力机制捕获长程依赖关系。

// 现象表现

- × 长序列预测效果远差于短序列，序列越长误差越大
- × 训练过程中梯度大部分时间接近 0，参数更新极其缓慢
- × 模型只能记住最近几个时间步的信息，对早期输入"失忆"
- × 加深 LSTM 层数后效果反而下降，出现退化现象

// 排查步骤

- 01 检查输入序列长度，确认是否超过 LSTM 的有效记忆范围（通常几百步）
- 02 监控训练过程中的梯度范数，验证是否存在梯度消失问题
- 03 尝试使用 TCN 或 Transformer 架构，对比长序列预测效果
- 04 引入注意力机制（如 Bahdanau Attention、Multi-Head Attention）增强长程依赖捕获能力
- 05 考虑序列分段策略，结合滑动窗口或金字塔式多尺度建模

## MAPE在数据接近0时无限大

// 快速修复

使用 SMAPE（对称平均绝对百分比误差）或 MAE 替代 MAPE，避免分母为零或接近零，对不同量级数据分组评估。

// 现象表现

- × 预测指标波动极大，单个样本即可导致整体指标剧烈变化
- × 某些接近零的样本贡献了 99% 的误差，主导整体评估结果
- × 模型实际预测效果不错，但 MAPE 指标却差得离谱
- × 不同批次数据的 MAPE 缺乏可比性，受零值/小值影响严重

// 排查步骤

- 01 检查真实值中是否存在零值或接近零的数值，统计其占比
- 02 计算每个样本的绝对百分比误差，定位异常大的误差来源
- 03 改用 SMAPE 或 MAE 作为评估指标，重新评估模型性能
- 04 对数据按量级分组（如高、中、低区间），分别计算评估指标
- 05 考虑使用对数变换后的数据进行评估，或引入加权 MAPE
