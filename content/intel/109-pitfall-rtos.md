## FreeRTOS 任务优先级配置不当导致系统响应差

// 快速修复

为关键任务分配最高优先级，避免优先级翻转，使用互斥量实现优先级继承机制。

// 现象表现

- × 高优先级任务被低优先级任务阻塞
- × 系统响应不及时，实时性差
- × 关键事件得不到及时处理

// 排查步骤

- 01 检查所有任务的优先级配置，确保高优先级任务确实具有最高优先级
- 02 使用二进制信号量替代直接调用 vTaskDelay 或阻塞操作
- 03 配置优先级继承，在使用互斥量时启用 `configUSE_MUTEXES` 和 `configPRIORITY_INHERITANCE`

## 中断与任务通信不同步导致数据错乱

// 快速修复

在中断处理程序中仅标记事件（设置标志位），将业务逻辑放在任务中处理，使用队列解耦中断与任务。

// 现象表现

- × 中断处理中数据被破坏或丢失
- × 共享变量值不符合预期
- × 数据竞争导致计算结果错误

// 排查步骤

- 01 检查中断处理函数，确保仅执行最小化操作（如设置标志位）
- 02 在任务中处理业务逻辑，使用 `xTaskNotifyFromISR` 或 `xQueueSendFromISR`
- 03 对共享变量使用 `volatile` 修饰，并添加临界区保护 `taskENTER_CRITICAL`

## 临界区嵌套导致系统死锁

// 快速修复

避免在临界区中调用可能阻塞的API函数，尽量减少临界区范围，使用互斥量替代递归临界区。

// 现象表现

- × 系统完全卡死，无任何响应
- × 看门狗复位，系统反复重启
- × 任务CPU使用率为0

// 排查步骤

- 01 检查 `taskENTER_CRITICAL` 与 `taskEXIT_CRITICAL` 配对是否正确
- 02 检查代码是否存在递归进入临界区的情况
- 03 使用互斥量（`xSemaphoreCreateMutex`）替代直接临界区操作

## 内存分配失败导致系统崩溃

// 快速修复

使用静态内存分配替代动态分配，预分配固定大小内存池，运行时务必检查 `malloc` 返回值。

// 现象表现

- × `malloc` 或 `pvPortMalloc` 返回 NULL
- × 系统运行一段时间后随机崩溃
- × 内存碎片化导致分配失败

// 排查步骤

- 01 使用静态分配替代动态分配：`StaticTask_t`、`StaticQueue_t` 等结构
- 02 创建固定大小内存池，使用 `xStreamBufferCreateStatic` 或自定义内存池
- 03 在每次分配后检查返回值，确保 `NULL` 时有降级处理
